## Using the Hosted Version

The fastest way to get started is using the hosted version at [repolyze.ossium.live](https://repolyze.ossium.live).

<Steps>
  <Step title="Visit Repolyze">
    Navigate to [repolyze.ossium.live](https://repolyze.ossium.live) in your browser.
  </Step>
  
  <Step title="Enter Repository URL">
    Paste a GitHub repository URL into the input field:
    
    ```text
    https://github.com/vercel/next.js
    ```
    
    <Tip>
      You can analyze any public GitHub repository. Private repositories require authentication (coming soon).
    </Tip>
  </Step>
  
  <Step title="Click Analyze">
    Press the **Analyze** button or hit Enter. You'll see real-time progress as Repolyze:
    - Fetches repository metadata
    - Analyzes file structure
    - Examines dependencies
    - Generates insights with AI
    - Creates architecture diagrams
  </Step>
  
  <Step title="Explore Results">
    Review the comprehensive analysis:
    
    <Tabs>
      <Tab title="Overview">
        - **Overall Health Score** (0-100)
        - Repository metadata and statistics
        - Tech stack detection
        - AI-generated summary
      </Tab>
      
      <Tab title="Scores">
        - Code Quality
        - Documentation
        - Security
        - Maintainability
        - Test Coverage
        - Dependencies
      </Tab>
      
      <Tab title="Insights">
        - Strengths and weaknesses
        - Security concerns
        - Improvement suggestions
        - Best practice recommendations
      </Tab>
      
      <Tab title="Architecture">
        - Component relationship diagrams
        - Data flow visualizations
        - File tree structure
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Branch Analysis

Repolyze supports analyzing any branch, not just the default branch.

<Steps>
  <Step title="Enter Repository URL">
    Start by entering a repository URL and clicking analyze.
  </Step>
  
  <Step title="Select Branch">
    Once the initial analysis completes, click the **Branch Selector** dropdown in the header.
  </Step>
  
  <Step title="Choose Branch">
    Select a different branch from the list to re-analyze.
    
    <Note>
      Branch analysis counts toward your daily analysis limit.
    </Note>
  </Step>
</Steps>

## Rate Limits

Repolyze has tiered rate limits to ensure fair usage:

| Tier | Daily Analyses | Cost |
| --- | --- | --- |
| **Anonymous** | 1 per day | Free |
| **Free Account** | 3 per day | Free |
| **Pro Account** | 44 per day | Paid |

<Tip>
  Sign in with GitHub to unlock 3 free analyses per day. Upgrade to Pro for 44 analyses.
</Tip>

## Running Locally

For self-hosting or development, run Repolyze locally.

### Prerequisites

<AccordionGroup>
  <Accordion icon="node-js" title="Node.js 18+">
    Repolyze requires Node.js 18.0 or higher.
    
    ```bash
    # Check your version
    node --version
    ```
    
    If you need to install or upgrade Node.js, visit [nodejs.org](https://nodejs.org/).
  </Accordion>
  
  <Accordion icon="box" title="pnpm Package Manager">
    Repolyze uses pnpm for dependency management.
    
    ```bash
    # Install pnpm globally
    npm install -g pnpm
    
    # Verify installation
    pnpm --version
    ```
  </Accordion>
  
  <Accordion icon="database" title="PostgreSQL Database">
    Required for authentication and rate limiting.
    
    ```bash
    # Install PostgreSQL
    # macOS
    brew install postgresql@16
    
    # Ubuntu/Debian
    sudo apt install postgresql-16
    
    # Start PostgreSQL
    brew services start postgresql@16  # macOS
    sudo systemctl start postgresql    # Linux
    ```
  </Accordion>
</AccordionGroup>

### Installation

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/OssiumOfficial/Repolyze.git
    cd Repolyze
    ```
  </Step>
  
  <Step title="Install Dependencies">
    ```bash
    pnpm install
    ```
    
    <Note>
      This will also run `prisma generate` automatically to set up the database client.
    </Note>
  </Step>
  
  <Step title="Configure Environment">
    Copy the example environment file and add your API keys:
    
    ```bash
    cp .env.example .env.local
    ```
    
    Edit `.env.local` with your keys:
    
    <CodeGroup>
    ```bash Required Variables
    # OpenRouter API Key (get at: https://openrouter.ai/keys)
    OPENROUTER_API_KEY=sk-or-v1-your-api-key-here
    
    # GitHub Personal Access Token (get at: https://github.com/settings/tokens)
    # Required scopes: repo, read:user
    GITHUB_TOKEN=ghp_your-github-token-here
    
    # Database URL
    DATABASE_URL=postgresql://user:password@localhost:5432/repolyze
    
    # NextAuth Secret (generate with: npx auth secret)
    AUTH_SECRET=your-auth-secret-here
    ```
    
    ```bash Optional Variables
    # Site URL (for production)
    NEXT_PUBLIC_SITE_URL=http://localhost:3000
    
    # GitHub OAuth (for user authentication)
    AUTH_GITHUB_ID=your-github-oauth-client-id
    AUTH_GITHUB_SECRET=your-github-oauth-client-secret
    
    # Google OAuth (optional)
    AUTH_GOOGLE_ID=your-google-client-id
    AUTH_GOOGLE_SECRET=your-google-client-secret
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Set Up Database">
    Initialize the PostgreSQL database:
    
    ```bash
    # Create database
    createdb repolyze
    
    # Run migrations
    pnpm prisma migrate dev
    
    # Generate Prisma client (runs automatically on install)
    pnpm prisma generate
    ```
  </Step>
  
  <Step title="Start Development Server">
    ```bash
    pnpm dev
    ```
    
    Open [http://localhost:3000](http://localhost:3000) in your browser.
    
    <Tip>
      The development server supports hot reloading. Changes to code are reflected instantly.
    </Tip>
  </Step>
</Steps>

### Getting API Keys

<Tabs>
  <Tab title="GitHub Token">
    1. Go to [GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)](https://github.com/settings/tokens)
    2. Click **Generate new token (classic)**
    3. Give it a descriptive name (e.g., "Repolyze Local Dev")
    4. Select scopes:
       - ✅ `repo` (Full control of private repositories)
       - ✅ `read:user` (Read user profile data)
    5. Click **Generate token**
    6. Copy the token immediately (you won't see it again)
    7. Add to `.env.local` as `GITHUB_TOKEN=ghp_...`
    
    <Warning>
      Keep your token secure. Never commit it to version control.
    </Warning>
  </Tab>
  
  <Tab title="OpenRouter Key">
    1. Visit [OpenRouter](https://openrouter.ai/)
    2. Sign up or log in
    3. Navigate to **Settings → API Keys**
    4. Click **Create Key**
    5. Give it a name (e.g., "Repolyze")
    6. Copy the generated key
    7. Add to `.env.local` as `OPENROUTER_API_KEY=sk-or-v1-...`
    
    <Note>
      OpenRouter provides access to multiple AI models. Repolyze uses this for flexible model selection and better reliability.
    </Note>
  </Tab>
  
  <Tab title="Auth Secret">
    Generate a secure random string for NextAuth:
    
    ```bash
    npx auth secret
    ```
    
    This command generates a cryptographically secure random string and adds it to your `.env.local` file automatically.
    
    Alternatively, generate manually:
    
    ```bash
    openssl rand -base64 32
    ```
  </Tab>
</Tabs>

## Testing Your Setup

Verify everything is working:

<CodeGroup>
```bash Test Analyze Endpoint
curl -X POST http://localhost:3000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"repoUrl": "https://github.com/vercel/next.js"}'
```

```bash Test Branches Endpoint
curl "http://localhost:3000/api/branches?repo=vercel/next.js"
```

```bash Test Share Page
open http://localhost:3000/share/vercel/next.js
```
</CodeGroup>

<Check>
  If all endpoints return valid JSON responses, your setup is complete!
</Check>
